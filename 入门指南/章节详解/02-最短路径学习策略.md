# 第二章：最短路径学习策略（3-6 个月）

## 核心原则

**不走弯路 > 系统全面**。你的目标是用最短时间获得 AI 生产力，不是考试拿满分。

---

## 跳过的部分（省下 2-3 个月）

### 为什么跳过

传统 AI/ML 课程动辄从线性回归、SVM、决策树讲起，再到 CNN、RNN，最后才到 Transformer。这条路适合在校学生，不适合有工程经验的在职转型者。

### 具体跳过清单

| 可跳过的内容 | 原因 | 何时回头补 |
|------------|------|-----------|
| 经典机器学习算法（SVM、决策树、随机森林） | 对 LLM 应用帮助极小 | 做推荐系统时再看 |
| 严格的数学证明（收敛性证明、信息论推导） | 你需要直觉不需要证明 | 想发论文时 |
| CNN 图像分类详细原理 | 除非做多模态，否则不急 | 做视觉模型时 |
| RNN/LSTM 详细原理 | 已被 Transformer 取代 | 基本不需要 |
| 传统 NLP（词袋、TF-IDF、Word2Vec 训练） | LLM 时代直接用 Embedding API | 做检索优化时了解即可 |
| 强化学习全套理论 | 除非做 RLHF 或游戏强化学习 | 做 RLHF 调优时 |
| 完整的概率图模型 | 研究向内容 | 基本不需要 |

### 特别说明

这不是说这些知识"没用"，而是说对你**当前目标**（成为能产出 AI 应用的工程师）来说，ROI 太低。当你在实际项目中遇到需要这些知识的场景时，再按需学习效率更高。

## 每个阶段的学习深度定义

### 知识掌握程度标准

对于每个主题，明确你需要达到的层次：

| 层次 | 定义 | 本指南的要求 |
|------|------|-------------|
| L1 知道 | 知道这个东西存在，能说出大概是什么 | 所有主题至少 L1 |
| L2 理解 | 能解释原理，画出示意图 | 核心概念要到 L2 |
| L3 会用 | 能用现有工具/库实现功能 | 重点工具要到 L3 |
| L4 能改 | 能修改源码、定制实现 | 部署优化阶段要到 L4 |
| L5 能创 | 能设计新方案解决未见过的问题 | 项目实战阶段目标 |

### 各阶段深度要求

```
阶段1 Python:        L3（会用）
阶段2 深度学习:      L2-L3（理解原理 + 能写训练代码）
阶段3 Transformer:   L2-L4（理解原理 + 能读改 LLM 源码）
阶段4 LLM应用/微调:  L3-L4（会用 + 能定制）
阶段5 RAG/Agent:     L3-L5（会用 + 能设计新系统）
阶段6 推理部署:      L4-L5（你的主场，能深入定制优化）
```

## 时间分配策略

### 工作日策略（每天 1-1.5 小时）

```
通勤/午休：看视频、听播客、读文章（输入）
晚上：跑代码、做实验（实践）
```

**建议碎片化内容**：
- Karpathy YouTube 视频（可倍速）
- 知乎/公众号高质量技术文章
- HuggingFace Blog
- arXiv 论文摘要

### 周末策略（每天 2-3 小时）

```
周六上午：动手写代码，跑实验
周日上午：复盘总结，整理笔记
```

**建议整块时间内容**：
- 跟着教程写代码
- 改动开源项目跑实验
- 写技术博客总结

### 节奏管理

- **前 2 个月**：学习为主（70% 学 + 30% 练）
- **中间 2 个月**：平衡（50% 学 + 50% 练）
- **后 2 个月**：实践为主（30% 学 + 70% 练）

## 学习效率加速器

### 1. 利用你的工程师思维

```
不要：从理论推导开始
要：先跑通一个 Demo，再倒推原理

不要：追求完美理解再动手
要：边做边学，遇到不懂的记下来后续补

不要：从第一页看到最后一页
要：带着问题查阅，按需学习
```

### 2. 建立类比映射表

每学一个新概念，立刻找一个游戏后端的对应概念：

```
Tensor          ≈  多维数组 / Buffer
GPU             ≈  协处理器（类比你用过的 DSP/FPGA）
梯度下降        ≈  物理引擎的数值积分
Batch Size      ≈  批处理请求量
Epoch           ≈  全量数据遍历一遍
过拟合          ≈  过度针对测试服环境优化导致线上不适配
正则化          ≈  限制代码复杂度防止过度设计
Embedding       ≈  哈希映射到高维空间
Attention       ≈  "关注度"加权查询（类似加权路由）
Fine-tuning     ≈  二次开发/插件定制
RAG             ≈  "运行时动态加载配置"而非"硬编码"
```

### 3. 输出倒逼输入

给自己设定输出目标：
- 每周写 1 篇学习笔记（给未来的自己看）
- 每 2 周做 1 个小 Demo
- 每月写 1 篇总结文章（可发知乎/博客）
- 6 个月后完成 1 个完整项目

## 学习资源选择原则

### 好资源的特征

1. **工程师视角**（不是数学家视角）
2. **有可运行代码**（不是纯 PPT）
3. **循序渐进**（每一步都能跑出结果）
4. **实战导向**（教你做项目不只是教概念）

### 避免的资源类型

1. 纯理论的大部头教科书（如《统计学习方法》——好书但现在不适合你）
2. 过于入门的"AI 科普"内容
3. 已经过时的教程（2020 年以前的 NLP 教程）
4. 过于前沿的研究论文（等你有基础再看）

## 里程碑检查点

每个阶段结束，用这些问题自测：

### 阶段 1 完成标志
- [ ] 能用 NumPy 手写矩阵乘法
- [ ] 能用 Pandas 读取和处理数据文件
- [ ] 环境配置好（conda + Jupyter + GPU 驱动）

### 阶段 2 完成标志
- [ ] 能不看资料写出 PyTorch 训练循环
- [ ] 能解释反向传播在干什么（不必推公式）
- [ ] 跑通一个 MNIST 或文本分类任务

### 阶段 3 完成标志
- [ ] 能画出 Transformer 架构图并解释每个组件
- [ ] 手写过 nanoGPT 或类似实现
- [ ] 能读懂 LLaMA/Qwen 的模型代码

### 阶段 4 完成标志
- [ ] 能用 API 构建一个有用的 LLM 应用
- [ ] 成功微调过至少一个模型（哪怕 1B 参数）
- [ ] 理解 LoRA 在做什么

### 阶段 5 完成标志
- [ ] 搭建过一个完整的 RAG 系统
- [ ] 实现过一个能调用工具的 Agent
- [ ] 能解释 RAG vs 微调各自适用场景

### 阶段 6 完成标志
- [ ] 用 vLLM 部署过一个 LLM 服务
- [ ] 做过模型量化并对比效果
- [ ] 能说出推理优化的 3-5 个关键技术
