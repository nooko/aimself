## 每周时间投入建议

```
工作日（周一到周五）：
├── 通勤时间：听/看视频 (30min × 2 = 1h)
├── 午休时间：看技术文章 (30min)
└── 晚上：写代码/跑实验 (1h)
合计：~2.5h/天 × 5 = 12.5h

周末（灵活安排）：
├── 周六上午：动手项目 (3h)
└── 周日上午：复盘+笔记 (2h)
合计：~5h

每周总计：~15-17h（比原估计的 12h 多一些，但更现实）
```

## 弹性调整策略

### 如果进度快

- 跳过基础直接进入下一阶段
- 增加项目复杂度
- 尝试更多工具和框架
- 开始写技术博客

### 如果进度慢

- 减少广度，专注最核心内容
- 把"理解原理"降级为"会用就行"
- 跳过对比实验，只用推荐方案
- 延长到 7-8 个月也完全 OK

### 如果工作太忙

```
最低限度计划（每周 6h）：
- 工作日：每天 30min 看视频
- 周末：集中 3h 动手
- 延长到 9-12 个月完成

核心不能砍的：
1. PyTorch 训练循环（必须手写过）
2. Transformer 原理（必须理解 Attention）
3. LLM API 使用（必须跑通 Function Calling）
4. 至少一次微调实践
5. 至少部署过一次 LLM 服务
```
