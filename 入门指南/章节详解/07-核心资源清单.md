# 第七章：核心资源清单（精选 + 详细说明）

## 必看视频——详细介绍

### 1. Andrej Karpathy "Neural Networks: Zero to Hero"

**为什么强烈推荐**：
- Karpathy 是前 Tesla AI 总监、前 OpenAI 研究员
- **工程师视角**讲解，不是数学家视角
- 从零实现，每一行代码都有解释
- 循序渐进，无需任何 ML 背景

**系列内容（2023-2024）**：

| 序号 | 视频标题 | 内容 | 时长 | 优先级 |
|------|---------|------|------|--------|
| 1 | The spelled-out intro to language modeling | Bigram 模型入门 | 1.5h | ⭐⭐⭐ |
| 2 | Building makemore | 简单神经网络 | 1.5h | ⭐⭐⭐ |
| 3 | Building makemore (MLP) | 多层感知机 | 1.5h | ⭐⭐ |
| 4 | Building makemore (BatchNorm) | 训练技巧 | 1h | ⭐⭐ |
| 5 | Building makemore (WaveNet) | 高级架构 | 1h | ⭐ |
| 6 | **Let's build GPT** | **核心！手写 GPT** | 2h | ⭐⭐⭐⭐⭐ |
| 7 | Let's build the GPT tokenizer | Tokenizer 实现 | 2h | ⭐⭐⭐ |
| 8 | Let's reproduce GPT-2 | 完整训练流程 | 4h | ⭐⭐⭐ |

**学习建议**：
```
最低要求：看第 1、2、6 集（5 小时）
推荐：看 1-6 集（9 小时）
完整：全部看（15+ 小时）

观看方式：
1. 1.5x 倍速先过一遍
2. 暂停，自己先写代码
3. 对比 Karpathy 的实现
4. 修改代码做实验
```

**链接**：YouTube 搜索 "Andrej Karpathy Neural Networks: Zero to Hero"

## 必读项目——详细说明

### 1. nanoGPT

**仓库**：github.com/karpathy/nanoGPT

**为什么必读**：
- ~300 行代码实现完整 GPT
- 代码清晰无复杂抽象
- 可以直接改代码做实验
- 理解 GPT 架构的最短路径

**建议学习方式**：
```python
# 核心文件：
# model.py — GPT 模型定义（~150 行）
# train.py — 训练脚本（~150 行）

学习步骤：
1. 先读 model.py，对照 Karpathy 视频
2. 跑通训练（Shakespeare 数据集）
3. 改超参数观察效果（层数、头数、embedding 维度）
4. 用自己的数据训练（如游戏台词）
5. 理解生成过程
```

---

### 2. llama.cpp

**仓库**：github.com/ggerganov/llama.cpp

**为什么必读**：
- 纯 C/C++ 实现 LLM 推理
- 理解底层推理流程的最佳代码
- 量化实现（GGUF 格式）
- 你的 C++ 背景能直接读懂

**学习重点**：
```
├── llama.h/llama.cpp — 核心推理逻辑
├── ggml.h — 张量运算库
├── quantize.cpp — 量化实现
└── sampling.cpp — 采样策略

对你有特殊价值：
- 理解 KV-Cache 的底层实现
- 理解量化的具体操作
- C++ 工程化的范例（内存管理、性能优化）
- 学习 GGUF 格式
```

---

### 3. vLLM

**仓库**：github.com/vllm-project/vllm

**为什么必读**：
- 工业级推理引擎，生产环境标准选择
- PagedAttention 原始实现
- 学习高性能 AI 服务设计

**学习重点**：
```
├── engine/ — 推理引擎核心
│   ├── llm_engine.py — 引擎主逻辑
│   └── async_llm_engine.py — 异步引擎
├── core/
│   ├── scheduler.py — 请求调度器（重点！）
│   └── block_manager.py — KV-Cache 内存管理（重点！）
├── worker/ — GPU 工作进程
└── entrypoints/ — API 服务入口

对你有特殊价值：
- 请求调度策略（类似游戏服务器请求调度）
- 内存管理（PagedAttention ≈ 虚拟内存）
- 异步服务设计
- Continuous Batching 实现
```

---

### 4. LLaMA-Factory

**仓库**：github.com/hiyouga/LLaMA-Factory

**为什么必读**：
- 微调的"瑞士军刀"，支持 100+ 模型
- Web UI 零代码微调
- 涵盖 SFT/RLHF/DPO 全流程
- 中文文档完善

**学习重点**：
```
用法：
├── 用 Web UI 快速体验微调全流程
├── 学习数据集格式要求
├── 理解各种微调方法的配置差异
└── 对比不同超参数效果

代码学习：
├── src/llmtuner/model — 模型加载策略
├── src/llmtuner/train — 训练流程
└── src/llmtuner/data — 数据处理
```

**为什么了解**：
- Agent 设计模式参考

```
MetaGPT：

对你的价值：

- 状态管理 ≈ 游戏状态同步
```
